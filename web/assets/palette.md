# StackWeft Palette

## Core Colors

| Token | Hex | Usage |
| --- | --- | --- |
| Background | `#0f1115` | Primary app shell, SVG weave cutouts, and dark surfaces. |
| Accent blue | `#2b6cff` | Primary actions, links, selected states, and warp/weft highlight threads. |
| Success/active green | `#2ecc71` | Verified states, passing checks, live activity, and secondary weave threads. |
| Light text | `#e6e6e6` | Primary labels, wordmark text, and high-contrast UI copy on dark surfaces. |

## Usage Notes

Use `#0f1115` as the dominant canvas with restrained blue and green accents. Keep blue for intent/navigation emphasis and green for evidence, verification, and active delivery states. Avoid broad gradients and cartoon AI motifs; the visual language should feel like an enterprise developer tool.

## Typography

Recommended stack: `Inter`, `SF Pro Display`, `Segoe UI`, `PingFang SC`, `Noto Sans CJK SC`, `Microsoft YaHei`, sans-serif. Use medium-to-semibold weights for navigation and labels, with strong but compact wordmark weight.

## Spacing And Radius

Use a 4px spacing base with common steps at 8px, 12px, 16px, 24px, and 32px. Prefer tight, functional radii: 6px for controls, 8px for cards or panels, and fully rounded ends only for thread motifs, pills, or compact status indicators.
